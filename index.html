Proyecto de Investigación — Crear sitios web

Introducción

Los sitios web son espacios digitales donde se organiza y muestra información mediante páginas enlazadas. En este proyecto exploramos las herramientas y procesos para crear sitios web, comparando métodos visuales (plantillas) con programación desde cero. También aprenderemos conceptos básicos de diseño y accesibilidad para que una página sea útil para todo público.

Objetivos: comprender la estructura básica de una web, conocer herramientas accesibles y practicar un flujo de trabajo desde la idea hasta la publicación en línea.

Historia

La Web nació como proyecto de investigación en el CERN a finales de los años 80. Tim Berners-Lee propuso la idea en 1989 y en 1991 publicó el primer sitio web público. En los primeros años se trabajó con HTML puro; más tarde llegaron los estilos (CSS, mediados de los 90) y JavaScript (1995) que permitieron interactividad. Desde entonces la web ha pasado de páginas estáticas a complejas aplicaciones en línea.

Avances clave: aparición de navegadores gráficos (p. ej. Mosaic), estandarización por la W3C, y la llegada de frameworks y servicios que facilitan el desarrollo moderno.

Importancia

Los sitios web tienen un rol central en la vida diaria y profesional:

  • Educación: plataformas y materiales en línea.
  • Comercio: tiendas y catálogos digitales.
  • Comunicación: información institucional y redes.
  • Identidad digital: presencia personal o empresarial en Internet.

Aprender a crear páginas abre oportunidades laborales y mejora la capacidad de comunicar ideas de forma estructurada y accesible.

Tipos de sitios web

  • Blog: publicación periódica de artículos y opiniones.
  • Portafolio: muestra de trabajos personales o artísticos.
  • Tienda en línea (e-commerce): venta de productos o servicios.
  • Portal informativo: noticias, bases de datos o enciclopedias.
  • Red social / Comunidad: interacción entre usuarios.
  • Landing page: página de presentación para campañas o servicios.

Herramientas

Algunas herramientas útiles según tu nivel y objetivo:

  • Editor visual: Wix, Google Sites — fáciles y rápidos.
  • CMS: WordPress — ideal para blogs y sitios administrables.
  • Editores de código: VS Code, Sublime Text — para programar.
  • Control de versiones: Git + GitHub — para guardar y compartir código.
  • Diseño: Figma o Canva — para prototipos y mockups.
  • Hosting / Deploy: GitHub Pages, Netlify, Vercel — despliegue rápido.

Consejo: combina herramientas — por ejemplo, diseña en Figma, programa en VS Code y publica con Netlify.

Lenguajes y tecnologías

  • HTML: estructura (títulos, párrafos, listas, imágenes).
  • CSS: apariencia (colores, tipografías, disposición responsive).
  • JavaScript: interactividad (formularios, animaciones, peticiones).
  • Frameworks front-end: React, Vue, Angular (facilitan interfaces complejas).
  • Back-end y bases de datos: Node.js, PHP, Python; MySQL, MongoDB.

Si recién empezás, comenzá por HTML + CSS y luego agrega JavaScript paso a paso.

Proceso paso a paso

  1. Planificación: objetivo, público, contenidos y mapa del sitio (sitemap).
  2. Wireframe / prototipo: boceta la estructura y la navegación (puede ser en papel o Figma).
  3. Diseño visual: define paleta de colores, tipografías e imágenes.
  4. Desarrollo: crea los archivos (index.html, styles.css, script.js) o usa una plantilla.
  5. Pruebas: revisa en móviles y navegadores; prueba enlaces y formularios.
  6. Publicación: sube al hosting y configura dominio (si lo tenés).
  7. Mantenimiento: actualiza contenido, corrige errores y optimiza rendimiento.

Tip práctico: usa Live Server en VS Code para ver cambios al instante mientras editás.

Comparación: plantillas vs programar

Plantillas / Constructores (Wix, Google Sites)

  • Ventaja: muy rápidos y accesibles para principiantes.
  • Desventaja: limitación en personalización y dependencia de la plataforma.

Programar desde cero (HTML/CSS/JS)

  • Ventaja: control total y aprendizaje profundo.
  • Desventaja: curva de aprendizaje más pronunciada y requiere más tiempo.

CMS (WordPress) combina facilidad de uso con más opciones de personalización, ideal para sitios que crecen con el tiempo.

Curiosidades

  • El primer sitio web (info.cern.ch) sigue accesible y conserva la información histórica del nacimiento de la Web.
  • Hoy una gran parte del tráfico web proviene de dispositivos móviles, por eso el diseño responsivo es esencial.
  • Muchos sitios modernos combinan varias tecnologías: frontend (React/Vue), APIs y back-end (Node/Python).

Conclusión

Crear sitios web es una habilidad combinada de lógica y creatividad. Desde diseñar estructura y contenido hasta aplicar estilos y añadir interactividad, el proceso enseña pensamiento crítico y resolución de problemas. Este proyecto demuestra que con herramientas adecuadas y práctica cualquiera puede construir una presencia digital profesional.
